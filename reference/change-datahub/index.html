Change Data Hub

You can change the Data Hub of a fully provisioned device without needing to reset it. This allows the device to send its telemetry data to a new cloud platform seamlessly during operation.

For example, if the device was initially configured to send telemetry data to the ThingsBoard cloud, but you now want to switch to AWS Cloud in operation, follow these steps:

How to Change Data Hub?​

To change the Data Hub, follow the steps below.

  1. Login to KEYNOA. Create a new Data Hub for the required cloud service.

    Create a Thingsboard, or an Azure, or an AWS Data Hub by following their respective steps.

  1. Choose the device for which the Data Hub needs to be changed. Click on the device settings.

  1. Select the newly created Data Hub and save the settings. Wait for a minimum of 2 minutes.

  1. You will be prompted with a note when the change is successfully implemented.

  1. The Data Hub is changed on the fly. The device will start sending telemetry data to the new IoT cloud automatically. This enables you to move the device from one IoT cloud service to another seamlessly.
info

Data Hub hot swap is supported from 0.7.0 release onwards.

Please refer Update KEYNOA.engine for more information.
