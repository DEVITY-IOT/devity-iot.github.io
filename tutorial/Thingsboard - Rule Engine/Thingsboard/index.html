Thingsboard Setup

Initial Situation​

You are the IoT manager of a brewing and beverage factory. Since the products should only be stored at certain temperatures, it is essential for the entire operation to detect failures of the cooling systems in a timely manner. To do this, you want to use sensors and IoT gateway at all storage points.

In order to detect failures directly, you now use the Thingsboard Rule Chain. The aim of this tutorial is to define a rule that immediately detects failures of the cooling system.

Adapt Rule Chain​

  1. In Thingsboard Cloud go to Rule Chains section and open Root Rule Chain.

  2. Familiarise yourself with the rule chain. Focus only on the red area. Incoming messages are distributed via the switch. If a message contains telemetry data, it is stored. We now want to check the stored data.

  3. Create a new script via drag and drop. Enter a name and adapt the function as shown. Click on "Add". 

return msg.temperature < 35;
  1. Connect the "Save Timeseries" module with our script. You can connect the modules by dragging and dropping. Select the label Success and click on "Add".

  1. Create a "Create Alarm" block that triggers the alarm. Enter a name. The default code does not need to be changed.

  2. Create a "Clear Alarm" block that resolves the alarm. Enter a name. The default code does not need to be changed.

  3. Connect the new alarm modules to the Temperature Check. To do this, create two connections via drag and drop, as you have already done. Make sure that the conditions False and True are set as shown in the screenshot.

  4. Very Important: Click on the "Apply Changes" Button at the bottom right to save all changes.

Build Dashboard​

  1. After customising the rule chain, we build a dashboard to visualise the alarms. Navigate to the section "Dashboards". At the top right you will find the "+" to create a new dashboard. Enter Title and Description and hit "Add".

  2. Open the newly created dashboard.

  3. Click on the Edit icon at the bottom right to add a new widget.

  4. Click on "Add new Widget" to create a new widget.

Temperature Display​

  1. The first thing we want to do is create a temperature display. Select the widget type "Cards" for this.

  2. Choose Simple Card.

  3. Thingsboard uses aliases as data source. Therefore, for the first widget we need to create an alias. To do this, select on "Create a new one!" from the drop-down menu.

  4. Enter the alias information shown in the screenshot below. For the device select your device and click on "Add"

  5. Now you can choose the newly created alias as data source. Choose "temperature" as attribute. Click "Add". Congratulations, you have created your first widget. Hint: If you can't find the attribute temperature, the device hasn't sent data yet.

Temperature Graph​

  1. The second widget should display a graph showing the temperature curve. To do this, select "Add new widget" again via the button at the bottom right. Then select the widget type "Timeseires Line Chart".

  2. Insert the data shown below and click "Add".

Alarms Table​

  1. The third widget should now show the incoming alarms. Therefore, add a new widget, this time with the type "Alarm widgets".

  2. Choose "Alarms table".

  3. Enter the two fields marked below. Select the "Active" and "Cleared" state as well as your Entity alias.

Set Sending Interval​

  1. The fourth widget should enable you to adjust the sending interval of the device. Add a new widge with the type "Input widgets".

  2. Choose "Update shared integer attribute".

  3. Select the entity and start tying in sending_interval into the data keys field. Create that attribute by clicking the "A" icon. After that click on "Settings".

  4. Adjust the titel to "sending interval" and click on "Add".

  5. You can adjust the sizes of the widget acording to your preferences. Very Important: Do not forget to click "Apply Changes" at the bottom right. Otherwise your Dashboard won't be saved.

  1. You can adjust the device sending interval by typing in the widget and confiming with the checkmark. Soon you can recognice that the interval between messages has changed.

Check incoming alarms​

  1. If the temperature rises above 35°, a new entry should occur in the Alarm table. The rule we defined before triggers the action that creates a new alarm.

  2. If the temperature falls below 35°, the state of the alarm will change to "cleared".

  3. You can adapt the Dashboard according to your wishes. Insert widgets and explore the Thingsboard functionalities.

Success! You now have a digitalized brewing and beverage factory.
